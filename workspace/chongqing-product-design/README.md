# 重庆电商平台产品设计工具包

## 概述

这是一个基于重庆内外贸统一大市场平台的业务域驱动的产品设计工具包。它帮助产品经理和设计师快速完成需求分析、PRD编写和UI设计，确保产品设计的质量和一致性。

## 特点

- **业务域驱动**：根据业务域（买家域、商家域、服务商域、平台运营域）组织专家团队
- **知识库支撑**：内置业务域PRD知识库和UI设计规范，确保设计的专业性
- **标准化流程**：提供标准化的需求分析、PRD编写和UI设计流程
- **中文友好**：所有文档和输出均使用中文，方便阅读和优化

## 适用场景

- 电商平台产品设计
- 多业务域产品设计
- 需求分析和PRD编写
- UI布局和交互设计
- 产品设计规范制定

## 工作流程

```
用户需求
  ↓
需求分析师：需求分析
  ↓
识别业务域
  ↓
业务域专家：PRD编写（并行）
  - 买家域产品专家
  - 商家域产品专家
  - 服务商域产品专家
  - 平台运营域产品专家
  ↓
UI布局设计师：布局设计
  ↓
UI交互设计师：交互设计
  ↓
整合输出
  ↓
生成最终文档
```

## 使用方法

### 1. 基础使用

直接调用设计编排器，提供需求描述：

```
@设计编排器 我需要在买家后台增加一个批量导入询价单的功能。
```

设计编排器会自动：

1. 调度需求分析师进行需求分析
2. 识别涉及的业务域（买家域）
3. 调度买家域产品专家编写PRD
4. 调度UI设计师设计布局和交互
5. 整合输出，生成最终文档

### 2. 分步使用

如果需要更精细的控制，可以分步调用：

**步骤1：需求分析**

```
@需求分析师 请分析以下需求：我需要在买家后台增加一个批量导入询价单的功能。
```

**步骤2：PRD编写**

```
@买家域产品专家 根据需求分析文档，编写买家后台批量导入询价单功能的PRD。
```

**步骤3：UI设计**

```
@UI布局设计师 根据PRD，设计批量导入询价单页面的布局。
@UI交互设计师 根据PRD和布局设计，设计批量导入询价单的交互细节。
```

### 3. 多业务域需求

对于涉及多个业务域的需求，设计编排器会自动协调：

```
@设计编排器 我需要优化商家入驻流程，包括卖家端的入驻申请和运营后台的审核功能。
```

设计编排器会：

1. 识别涉及的业务域（商家域、平台运营域）
2. 并行调度商家域产品专家和平台运营域产品专家
3. 检查两个PRD的一致性
4. 调度UI设计师设计布局和交互
5. 整合输出

## 最佳实践

1. **明确需求**：在开始设计前，确保需求清晰明确
2. **参考现有PRD**：参考现有PRD的格式和内容，保持一致性
3. **遵循规范**：严格遵循PRD标准模板和UI设计规范
4. **分步进行**：对于复杂需求，建议分步进行，逐步完善
5. **及时沟通**：遇到问题及时与需求方沟通澄清
6. **质量检查**：完成后进行质量检查，确保文档质量

## 注意事项

1. 所有文档使用中文编写
2. 遵循PRD标准模板和UI设计规范
3. 参考现有PRD，保持格式和内容的一致性
4. 对于跨业务域的需求，注意协调和一致性检查
5. UI设计需要考虑响应式和无障碍设计
