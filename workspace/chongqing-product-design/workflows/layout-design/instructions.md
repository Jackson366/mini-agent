# UI布局设计工作流指南

## 步骤1：需求理解

**输入**：PRD文档

**活动**：
1. 阅读PRD，理解功能需求
2. 识别页面类型（列表页、表单页、详情页等）
3. 识别页面元素（表单、表格、卡片等）
4. 识别布局约束（屏幕尺寸、设备类型等）

**输出**：页面类型清单、页面元素清单

## 步骤2：布局设计

**输入**：页面类型清单、页面元素清单

**活动**：
1. 选择合适的布局类型
2. 规划页面元素的位置
3. 设计信息层级
4. 设计响应式规则

**输出**：页面布局说明、布局示意图

## 步骤3：输出文档

**活动**：
1. 按照模板格式输出布局设计文档
2. 保存到指定输出目录

## 布局设计原则

1. **清晰性**：布局清晰，信息层级分明
2. **一致性**：保持布局的一致性
3. **平衡性**：元素分布平衡，视觉舒适
4. **对齐性**：元素对齐，整齐有序
5. **留白性**：合理留白，避免拥挤
6. **响应性**：适配不同屏幕尺寸

## 栅格系统

- 采用12栅格系统
- 列间距：20px
- 外边距：40px（大屏）、20px（中屏）、10px（小屏）

## 布局类型参考

### 后台管理系统布局
- 顶部导航 + 左侧菜单 + 主内容区 + 页脚
- 适用于：卖家后台、服务商后台、运营后台

### 前台页面布局
- 顶部导航 + 主内容区 + 页脚
- 适用于：买家端、服务市场、平台门户

### 列表页布局
- 查询区域 + 操作区域 + 列表区域 + 分页区域

### 表单页布局
- 表单标题 + 表单内容 + 操作按钮

### 详情页布局
- 页面标题 + 操作按钮 + 信息卡片
