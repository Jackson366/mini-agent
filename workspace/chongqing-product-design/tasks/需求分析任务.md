# 需求分析任务

## 任务目标

对用户提出的需求进行深入分析，识别核心业务场景，明确需求范围和边界，为后续的PRD编写提供清晰的输入。

## 执行角色

需求分析师

## 任务步骤

### 1. 需求收集

**目标**：理解用户提出的需求

**活动**：

1. 仔细阅读用户提供的需求描述
2. 识别需求的核心目标
3. 识别需求的背景和动机
4. 识别需求的约束条件

**输出**：

- 需求概述
- 核心目标
- 背景说明
- 约束条件清单

### 2. 需求分析

**目标**：深入分析需求，识别业务场景和功能模块

**活动**：

1. 识别涉及的业务域（买家域、商家域、服务商域、平台运营域）
2. 识别涉及的用户角色
3. 分析业务场景和用户故事
4. 识别功能需求和非功能需求
5. 分析需求的依赖关系
6. 评估需求的复杂度和风险

**输出**：

- 涉及的业务域清单
- 用户角色清单
- 业务场景描述
- 功能需求清单
- 非功能需求清单
- 依赖关系图
- 风险清单

### 3. 需求澄清

**目标**：澄清模糊的需求，确认需求理解的准确性

**活动**：

1. 列出需要澄清的问题
2. 提供可能的理解和选项
3. 请用户确认或补充信息
4. 记录澄清结果

**输出**：

- 需求澄清问题清单
- 用户反馈
- 澄清结果

### 4. 需求确认

**目标**：与用户确认需求理解的准确性

**活动**：

1. 整理需求分析结果
2. 向用户确认需求理解
3. 收集用户反馈
4. 修改需求分析文档

**输出**：

- 确认的需求分析文档

## 注意事项

1. **保持客观**：客观分析需求，不带个人偏见
2. **主动提问**：对不清楚的需求主动提问
3. **关注价值**：始终关注需求的用户价值和业务价值
4. **控制范围**：明确需求边界，避免范围蔓延
5. **及时沟通**：遇到问题及时与用户沟通
6. **文档化**：所有分析结果都应文档化
