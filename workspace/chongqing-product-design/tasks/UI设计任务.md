# UI设计任务

## 任务目标

根据PRD文档，设计页面布局和交互细节，确保界面美观、易用、符合设计规范。

## 执行角色

- UI布局设计师：负责页面布局设计
- UI交互设计师：负责交互细节设计

## 任务步骤

### 阶段1：布局设计（UI布局设计师）

#### 1. 需求理解

**目标**：理解PRD文档，明确页面设计需求

**活动**：

1. 阅读PRD文档
2. 识别页面类型（列表页、表单页、详情页等）
3. 识别页面元素（表单、表格、卡片等）
4. 识别布局约束（屏幕尺寸、设备类型等）

**输出**：

- 页面类型清单
- 页面元素清单

#### 2. 布局设计

**目标**：设计页面整体布局

**活动**：

1. 选择合适的布局类型
2. 规划页面元素的位置
3. 设计信息层级
4. 设计响应式规则
5. 参考UI设计规范

**输出**：

- 页面布局说明
- 布局示意图（ASCII图或文字描述）

#### 3. 布局评审

**目标**：评审布局设计，确保质量

**活动**：

1. 自我评审布局设计
2. 检查布局的合理性
3. 检查布局的一致性
4. 修改布局设计

**输出**：

- 评审后的布局说明

### 阶段2：交互设计（UI交互设计师）

#### 1. 需求理解

**目标**：理解PRD文档和布局设计，明确交互设计需求

**活动**：

1. 阅读PRD文档
2. 理解布局设计方案
3. 识别交互场景
4. 识别交互元素

**输出**：

- 交互场景清单
- 交互元素清单

#### 2. 交互设计

**目标**：设计交互细节

**活动**：

1. 设计用户操作流程
2. 设计交互细节
3. 设计反馈和提示
4. 设计异常处理
5. 参考UI设计规范

**输出**：

- 交互说明文档
- 交互流程图

#### 3. 交互评审

**目标**：评审交互设计，确保质量

**活动**：

1. 自我评审交互设计
2. 检查交互的合理性
3. 检查交互的一致性
4. 修改交互设计

**输出**：

- 评审后的交互说明

## 输出模板

### 布局设计文档模板

```markdown
# [页面名称] 布局设计

## 页面类型

[列表页/表单页/详情页/其他]

## 布局类型

[后台管理系统布局/前台页面布局/其他]

## 页面结构

\`\`\`
+--------------------------------------------------+
| 顶部导航区域 |
+----------+---------------------------------------+
| | |
| 左侧 | 主内容区域 |
| 菜单 | |
| 区域 | |
| | |
+----------+---------------------------------------+
| 页脚区域 |
+--------------------------------------------------+
\`\`\`

## 区域说明

### 区域1：顶部导航区域

- 位置：页面顶部
- 尺寸：高度60-80px，宽度100%
- 内容：Logo、平台名称、语言切换、用户信息

### 区域2：左侧菜单区域

- 位置：页面左侧
- 尺寸：宽度200-240px，高度自适应
- 内容：一级菜单、二级菜单

### 区域3：主内容区域

- 位置：页面中间
- 尺寸：宽度自适应，最小宽度1000px，高度自适应
- 内容：根据功能模块设计

### 区域4：页脚区域

- 位置：页面底部
- 尺寸：高度60-80px，宽度100%
- 内容：平台信息、版权信息

## 响应式规则

- 大屏（≥1440px）：正常显示
- 中屏（1024px - 1439px）：左侧菜单可收起
- 小屏（768px - 1023px）：左侧菜单默认收起
- 移动端（<768px）：左侧菜单隐藏，通过按钮打开

## 组件清单

- 顶部导航组件
- 左侧菜单组件
- 表单组件
- 表格组件
- 按钮组件
- 输入框组件
- 下拉选择框组件
- 分页组件
```

### 交互设计文档模板

```markdown
# [页面名称] 交互设计

## 交互流程

\`\`\`
用户进入页面
↓
查看列表
↓
输入查询条件
↓
点击搜索按钮
↓
显示查询结果
↓
点击操作按钮
↓
执行操作
↓
显示操作结果
\`\`\`

## 交互细节

### 交互场景1：查询列表

- 触发条件：用户进入页面或点击搜索按钮
- 操作步骤：
  1. 用户输入查询条件
  2. 用户点击搜索按钮
  3. 系统显示加载动画
  4. 系统查询数据
  5. 系统显示查询结果
- 交互反馈：显示加载动画，查询完成后隐藏
- 页面变化：列表区域显示查询结果

### 交互场景2：提交表单

- 触发条件：用户点击提交按钮
- 操作步骤：
  1. 用户填写表单
  2. 用户点击提交按钮
  3. 系统验证表单
  4. 如果验证失败，显示错误提示
  5. 如果验证成功，显示加载动画
  6. 系统提交表单
  7. 系统显示成功提示
- 交互反馈：显示加载动画，提交完成后显示成功提示
- 页面变化：表单提交成功后，可能跳转到其他页面或刷新当前页面

## 交互元素

### 元素1：搜索按钮

- 类型：主要按钮
- 默认状态：蓝色背景，白色文字
- 悬停状态：背景色变浅
- 点击行为：触发查询操作
- 禁用状态：灰色背景，不可点击

### 元素2：输入框

- 类型：文本输入框
- 默认状态：白色背景，灰色边框
- 聚焦状态：蓝色边框
- 输入行为：用户可以输入文字
- 错误状态：红色边框，下方显示错误提示

## 反馈和提示

### 成功提示

- 触发条件：操作成功
- 提示内容：操作成功的消息
- 显示方式：页面顶部居中显示消息提示
- 持续时间：3秒后自动消失

### 错误提示

- 触发条件：操作失败或验证失败
- 提示内容：错误原因和建议
- 显示方式：页面顶部居中显示消息提示或输入框下方显示错误信息
- 持续时间：5秒后自动消失或用户关闭

### 加载提示

- 触发条件：正在执行操作
- 提示内容：加载动画
- 显示方式：按钮显示加载图标或页面显示加载动画
- 持续时间：操作完成后消失

## 异常处理

### 异常情况1：网络错误

- 处理方式：显示错误提示，提示用户检查网络连接
- 提示信息：网络连接失败，请检查网络后重试

### 异常情况2：服务器错误

- 处理方式：显示错误提示，提示用户稍后重试
- 提示信息：服务器繁忙，请稍后重试

### 异常情况3：权限不足

- 处理方式：显示错误提示，提示用户联系管理员
- 提示信息：您没有权限执行此操作，请联系管理员
```

## 注意事项

1. **遵循规范**：严格遵循UI设计规范
2. **保持一致**：保持布局和交互的一致性
3. **用户体验**：优先考虑用户体验
4. **响应式设计**：考虑不同屏幕尺寸的布局
5. **及时反馈**：提供及时的交互反馈
6. **容错设计**：设计容错机制，防止误操作
7. **无障碍设计**：考虑无障碍设计
8. **使用中文**：所有设计文档使用中文编写
