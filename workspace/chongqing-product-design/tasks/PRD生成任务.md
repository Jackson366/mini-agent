# PRD生成任务

## 任务目标

根据需求分析文档，由对应的业务域专家编写详细的PRD文档，包括功能需求、业务规则、交互说明、验收标准等。

## 执行角色

根据涉及的业务域选择对应的专家：

- 买家域产品专家
- 商家域产品专家
- 服务商域产品专家
- 平台运营域产品专家

## 任务步骤

### 1. 需求理解

**目标**：理解需求分析文档，明确PRD编写范围

**活动**：

1. 阅读需求分析文档
2. 识别涉及的功能模块
3. 分析功能的业务价值
4. 识别功能的依赖和约束
5. 与需求分析师澄清疑问

**输出**：

- 需求理解摘要
- 功能模块清单
- 疑问清单

### 2. 功能设计

**目标**：设计功能的详细方案

**活动**：

1. 设计功能流程
2. 设计页面布局
3. 设计交互细节
4. 定义字段和规则
5. 设计异常处理
6. 参考对应业务域知识库和现有PRD

**输出**：

- 功能流程图
- 页面布局说明
- 交互说明
- 字段定义
- 业务规则
- 异常处理方案

### 3. PRD编写

**目标**：按照PRD标准模板编写PRD文档

**活动**：

1. 按照PRD标准模板组织内容
2. 编写项目背景与目标
3. 编写需求详情
4. 提供功能清单
5. 编写详细功能描述
6. 定义验收标准
7. 规划交付时间

**输出**：

- PRD文档初稿

### 4. PRD评审

**目标**：评审PRD文档，确保质量

**活动**：

1. 自我评审PRD文档
2. 检查PRD的完整性
3. 检查PRD的准确性
4. 检查PRD的一致性
5. 修改PRD文档

**输出**：

- 评审后的PRD文档

## 输出模板

使用 `data/PRD标准模板.md` 作为输出模板。

## PRD编写规范

### 标题格式

```
【PRD-待评审】重庆项目 [端侧名称] [功能名称] V1.0
```

示例：

- 【PRD-待评审】重庆项目 买家端 批量导入询价单 V1.0
- 【PRD-待评审】重庆项目 商家端 商品批量导入优化 V1.0

### 功能描述格式

```markdown
### [功能名称]

#### 业务价值

[为什么需要这个功能，解决什么问题]

#### 功能描述

[功能做什么，怎么做]

#### 业务流程

[描述业务流程，可使用流程图]

#### 用户场景

**场景 1：[场景名]**

- 前置条件：[条件]
- 操作步骤：
  1. [步骤 1]
  2. [步骤 2]
  3. [步骤 3]
- 预期结果：[结果]

#### 页面布局

[描述页面布局，可附原型图]

#### 字段说明

| 字段名 | 类型 | 必填 | 校验规则 | 说明 |
| ------ | ---- | ---- | -------- | ---- |
| 字段1  | 文本 | 是   | 规则     | 说明 |

#### 业务规则

- 规则 1
- 规则 2

#### 交互说明

- 点击[按钮]：[行为]
- 页面跳转：[跳转逻辑]
- 提示信息：[提示内容]

#### 异常处理

- 异常情况 1：[处理方式]
- 异常情况 2：[处理方式]

#### 验收标准

- [ ] [标准 1]
- [ ] [标准 2]
```

### 表格使用规范

使用Markdown表格描述：

- 功能清单
- 字段说明
- 状态流转
- 权限矩阵

示例：

```markdown
| 序号 | 模块     | 功能           | 功能描述说明                   | 详细需求链接 |
| ---- | -------- | -------------- | ------------------------------ | ------------ |
| 1    | 询价管理 | 批量导入询价单 | 支持通过Excel批量导入询价单    | 见5.1        |
| 2    | 询价管理 | 询价单列表     | 显示所有询价单，支持查询和筛选 | 见5.2        |
```

### 原型图规范

- 提供交互链接（如有）
- 关键页面截图
- 标注说明

8. **使用中文**：所有PRD文档使用中文编写
