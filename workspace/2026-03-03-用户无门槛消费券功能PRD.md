## 一、项目背景与目标

### 1.1 业务背景

平台当前面临用户增长放缓、新用户首单转化率偏低（约15-20%）、沉睡用户召回困难等挑战。无门槛消费券作为一种简单直接的促销工具，能够有效降低用户决策门槛，提升下单转化率。

**核心问题**：
- 新用户首单转化率仅15-20%，低于行业平均水平（25%）
- 沉睡用户（30-90天未下单）召回率低于5%
- 价格敏感型用户对优惠规则复杂的券型（如满减券）参与度低

**解决思路**：
推出无门槛消费券，用户领取后可在下单时直接抵扣，无需满足任何消费金额条件。通过"新人券自动发放"提升首单转化，通过"沉睡用户召回券"激活老用户，通过"常态化签到券"培养用户购买习惯。

### 1.2 项目目标

#### 短期目标（MVP上线后3个月内）

| 指标 | 当前值 | 目标值 | 提升幅度 |
|-----|-------|-------|---------|
| **新用户首单转化率** | 15-20% | ≥25% | +25% |
| **沉睡用户召回率** | <5% | ≥8% | +60% |
| **券核销率** | - | 30-50% | - |
| **券ROI** | - | ≥3.0 | - |

#### 长期目标（6-12个月）

- 培养用户常态化购买习惯，降低"等券"心理依赖
- 建立梯度分摊机制，商家参与度≥60%
- 形成完善的风控体系，羊毛党拦截率≥95%

### 1.3 项目价值

**对用户**：
- 降低尝试门槛，新用户能够低风险体验平台服务
- 简单直接的优惠，无需计算凑单金额
- 即时满足感，领取即可使用

**对商家**：
- 获取新客流量，尤其是新用户和高价值用户
- 平台流量扶持，承担成本越高流量倾斜越大
- 灵活的参与机制，可选择是否参与及承担比例

**对平台**：
- 提升核心转化指标（首单转化率、召回率）
- 积累用户行为数据，为后续精细化运营打基础
- 建立商家补贴生态，平台可持续运营

## 二、用户角色与场景

### 2.1 用户角色定义

| 角色名称 | 角色描述 | 核心诉求 | 使用频率 |
|---------|---------|---------|---------|
| **新用户** | 注册≤7天，未完成首单 | 低风险尝试平台商品 | 一次性（新人券） |
| **沉睡用户** | 30-90天未下单 | 被优惠刺激重新下单 | 每90天1次 |
| **活跃用户** | 近7天有下单 | 额外福利，增加购买频次 | 每周≤1次 |
| **价格敏感型** | 对优惠高度敏感 | 追求性价比，降低消费成本 | 高频（签到券） |
| **平台运营** | 负责券发放策略和效果监控 | 控制成本，提升ROI | 每日监控 |
| **商家** | 提供商品和服务 | 获取流量，控制成本 | 每日查看数据 |

### 2.2 典型用户场景

#### 场景1：新人首次领券下单

**角色**：新用户小王
**背景**：小王首次注册平台，对商品质量不信任，不想花太多钱尝试

**用户旅程**：
```
1. 小王注册成功，平台自动发放10元无门槛新人券
2. 首页弹窗展示"新人礼包：10元无门槛券已到账"
3. 小王浏览商品，看到某商品标价39元
4. 加入购物车，进入结算页
5. 系统自动应用10元券，显示"实付29元，已省10元"
6. 小王支付29元，完成首单
```

**关键体验点**：
- 券到账强反馈（弹窗+动画+Toast）
- 自动应用券，降低决策成本
- 明确显示节省金额，强化获得感

#### 场景2：沉睡用户召回

**角色**：沉睡用户小李
**背景**：小李60天未下单，平台希望激活他

**用户旅程**：
```
1. 小李收到Push/短信："您有1张5元券即将过期，点击查看"
2. 点击进入领券中心，看到5元无门槛券
3. 点击领取，Toast提示"领取成功"
4. 系统推荐适合用券的商品（30-50元价位）
5. 小李选中商品，下单用券
```

**关键体验点**：
- 制造紧迫感（"即将过期"）
- 推荐匹配商品，降低浏览成本
- 短有效期（5天）促立即行动

#### 场景3：用户退款场景

**角色**：用户小张
**背景**：小张用5元券购买39元商品，实际支付34元，因质量问题申请退款

**决策点（Q1: 已选择方案A - 分场景返还）**：

```
场景A：未发货前退款
┌────────────────────────────────────┐
│ 退款金额：34元（实付金额）          │
│ 券处理：返还5元券到账户             │
│ 有效期：恢复原有效期（不延长）      │
│ 提示："退款成功，5元券已返还"       │
└────────────────────────────────────┘

场景B：已发货/已收货后退款
┌────────────────────────────────────┐
│ 退款金额：34元（实付金额）          │
│ 券处理：不返还，券已消耗            │
│ 提示："退款成功，券已使用无法返还" │
└────────────────────────────────────┘
```

**设计理由**：
- 未发货：用户未享受商品，返还券保障用户权益
- 已发货：用户已体验商品，不返还可防止恶意套利

#### 场景4：商家参与设置

**角色**：商家老板
**背景**：商家希望参与无门槛券活动获取流量，但担心成本

**用户旅程**：
```
1. 登录商家后台，进入"营销中心-无门槛券"
2. 看到"是否参与"开关（默认关闭）
3. 选择"参与"后，选择承担比例：
   ○ 不参与（平台100%承担，无流量扶持）
   ○ 承担10%（平台90%，流量扶持1.5倍）
   ○ 承担30%（平台70%，流量扶持2倍）  ← 选择此项
   ○ 承担50%（平台50%，流量扶持3倍）
   ○ 承担100%（流量扶持5倍）
4. 保存设置，开始接收流量
5. 每日查看后台数据：今日用券158张，承担474元
```

**关键设计（Q2: 已选择方案B - 梯度分摊）**：
- 商家完全自主选择，默认0%承担（平台100%）
- 承担比例越高，流量扶持越大
- 实时数据透明，清楚知道承担成本

### 4.2 用户分层发券策略

#### 策略配置（Q3: 已选择方案B - 分层有效期）

| 用户类型 | 面额 | 有效期 | 发放方式 | 限领规则 | 目标 |
|---------|-----|-------|---------|---------|------|
| **新用户** | 10元 | 7天 | 注册后自动发放 | 单手机号1张 | 首单转化率≥25% |
| **沉睡用户** | 5-8元 | 5天 | Push/短信召回链接，主动领取 | 每90天1张 | 召回率≥8% |
| **活跃用户** | 3元或随机1-5元 | 3天 | 签到/任务达成后发放 | 每周≤1张 | 培养习惯 |
| **价格敏感型** | 1-2元 | 1天 | 每日签到 | 每天1张 | 提升DAU |

**有效期调整说明**：
- 原方案沉睡用户3天 → 调整为5天（确保用户有足够时间使用）
- 原方案活跃用户24小时 → 调整为3天（避免过短导致高过期率）
- 所有类型统一"提前1天+1小时"双提醒机制

### 4.3 券核销流程

#### 4.3.1 结算页券自动应用

**流程图**：
```
用户进入结算页
      ↓
系统查询用户所有可用券
      ↓
过滤出适用当前订单的券
      ↓
┌─────────────────────┐
│ 是否有可用券？      │
└─────────────────────┘
      ↓ 是              ↓ 否
自动选择面额最大的券   显示"暂无可用券"
      ↓
显示:"已自动选择X元券(省最多)"
      ↓
提供"查看其他券"入口
      ↓
用户可选择切换其他券
      ↓
实时更新订单金额
```

**交互设计**：
```
┌──────────────────────────────────────┐
│ 订单金额：¥39.00                      │
│ ──────────────────────────────────── │
│ 优惠券：-¥5.00  [更换]               │
│ ✅ 已自动选择5元券（省最多）          │
│ ──────────────────────────────────── │
│ 实付金额：¥34.00                     │
│                                      │
│ [提交订单]                           │
└──────────────────────────────────────┘
```

**关键规则（Q4: 已选择方案A - 不支持叠加）**：
- MVP不支持券叠加
- 多张券时，仅能使用1张
- 系统自动选择面额最大的券
- 用户可手动切换到其他券

#### 4.3.2 订单金额计算逻辑

**正常订单计算**：
```
订单金额: 100元
- 无门槛券: -10元
= 用户实付: 90元

成本分摊（假设商家承担30%）:
券面额10元 = 平台7元 + 商家3元
```

**部分退款计算**：
```
原订单: 商品A(60元) + 商品B(40元) = 100元
用券: 10元无门槛券
实付: 90元

用户仅退款商品A:
退款金额 = 60 ÷ 100 × 90 = 54元
券处理 = 返还6元券给用户（按比例返还）
```

### 4.4 退款券返还规则

#### 决策规则（Q1: 方案A - 分场景返还）

| 退款场景 | 退款金额 | 券处理 | 提示文案 | 理由 |
|---------|---------|-------|---------|------|
| **未发货退款** | 实付金额 | 返还券，恢复原有效期 | "退款成功，5元券已返还，有效期至X月X日" | 用户未享受商品，保障权益 |
| **已发货未收货** | 实付金额 | 返还券，恢复原有效期 | "退款成功，5元券已返还" | 用户未确认收货 |
| **已收货退款** | 实付金额 | 不返还券 | "退款成功，券已使用无法返还" | 用户已体验商品 |
| **券即将过期时退款** | 实付金额 | 返还券但不延长有效期 | "退款成功，券已返还，有效期至X日X点（即将过期）" | 防止恶意延长有效期 |
| **部分退款（拆单）** | 按比例退款 | 返还按比例计算的券 | "退款成功，已返还X元券" | 保障用户权益 |

**关键逻辑**：
- 券返还后，有效期从原到期时间计算，**不延长**
- 如果券已过期，则不返还，提示"券已过期，无法返还"
- 防止用户通过"反复下单退款"来延长券有效期（风控监控）

### 10.3 关键假设

以下假设需要在项目启动前确认：

1. **用户规模假设**：
   - 新用户日增长：约1,000人
   - 沉睡用户池：约50,000人
   - 活跃用户：约200,000人

2. **成本预算假设**（Q5: 动态调整）：
   - MVP月预算起点：50万元
   - 动态调整规则：ROI≥3时增加预算，ROI<2时减少
   - 熔断机制：单月预算上限100万元

3. **技术假设**：
   - 设备指纹服务：接入第三方服务商
   - 消息推送：自有Push系统+短信服务商
   - 数据仓库：支持实时计算（5分钟延迟）
