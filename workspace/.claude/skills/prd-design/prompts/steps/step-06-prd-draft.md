# 步骤 06：PRD 草稿

## 强制执行规则（首先阅读）

### 通用规则
- 🛑 绝不在没有用户输入的情况下生成内容
- 📖 关键：在采取任何行动之前阅读完整的步骤文件
- 🔄 关键：使用 'C' 加载下一步时，确保读取整个文件
- 📋 你是引导者，不是内容生成器
- ✅ 你必须始终用中文输出

## 执行协议
- 🎯 在采取任何行动之前显示你的分析
- 💾 仅在用户选择 C（继续）时保存
- 📖 更新 frontmatter，将此步骤添加到 stepsCompleted
- 🚫 禁止在 C 被选择之前加载下一步

## 🚨 系统成功/失败指标

### ✅ 成功
- 菜单正确显示且用户输入正确处理
- 在继续之前更新 frontmatter
- 在执行之前读取整个下一步文件

### ❌ 系统失败
- 在用户未选择 'C'（继续）的情况下继续
- 未用步骤完成更新 frontmatter
- 仅读取部分步骤文件

---

## 步骤目标

协作创建 PRD 的核心章节：
1. 用户故事/用例
2. 详细功能描述
3. 功能清单
4. 非功能需求
5. 成功指标

## 执行指令

### 1. 读取当前 PRD 状态

使用 Read 工具读取完整的 PRD 文档。

回顾已填写的内容：
- 项目背景与目标
- 领域特定要求（如果有）

### 2. 用户故事/用例

与用户协作定义用户故事：

```
让我们定义核心用户故事。

对于每个主要用户角色，描述他们的典型使用场景：

角色 1：[用户角色名称]
- 作为 [角色]
- 我想要 [目标]
- 以便 [价值]

请描述第一个用户故事。
```

收集 3-5 个核心用户故事。

将用户故事添加到 PRD 文档：

```markdown
### 5.1 用户故事

#### 故事 1：[标题]
- 作为 [角色]
- 我想要 [目标]
- 以便 [价值]

验收标准：
- [ ] [标准 1]
- [ ] [标准 2]
```

### 3. 功能清单

与用户协作列出所有功能：

```
基于用户故事，让我们列出需要实现的功能。

功能应该按模块组织。例如：
- 用户管理模块
- 订单管理模块
- 支付模块

请告诉我第一个模块和它包含的功能。
```

更新功能清单表格：

```markdown
### 功能清单

| 序号 | 模块 | 功能 | 功能描述说明 | 优先级 |
| ---- | ---- | ---- | ------------ | ------ |
| 1    | 用户管理 | 用户注册 | 支持邮箱/手机号注册 | P0 |
| 2    | 用户管理 | 用户登录 | 支持多种登录方式 | P0 |
```

### 4. 详细功能描述

对于每个核心功能（P0/P1），协作编写详细描述：

```
让我们详细描述功能：[功能名称]

请提供：
1. 功能概述
2. 输入/输出
3. 业务规则
4. 异常处理
5. 界面要求（如果有）
```

将详细描述添加到文档：

```markdown
#### 1. 用户注册

**功能概述**
[用户提供的描述]

**输入**
- [输入项 1]
- [输入项 2]

**输出**
- [输出项 1]

**业务规则**
- [规则 1]
- [规则 2]

**异常处理**
- [异常场景 1]：[处理方式]

**界面要求**
- [要求 1]
```

### 5. 非功能需求

与用户讨论非功能需求：

```
让我们定义非功能需求：

性能要求：
- 响应时间：[例如：页面加载 < 2 秒]
- 并发用户：[例如：支持 10000 并发]

安全要求：
- 数据加密：[要求]
- 访问控制：[要求]

可用性要求：
- 系统可用性：[例如：99.9%]
- 容灾备份：[要求]

请提供你的非功能需求。
```

添加非功能需求章节：

```markdown
## 5.6 非功能需求

### 性能要求
[用户提供的要求]

### 安全要求
[用户提供的要求]

### 可用性要求
[用户提供的要求]
```

### 6. 成功指标

与用户定义成功指标：

```
如何衡量这个产品的成功？

请提供可量化的指标，例如：
- 用户增长：[目标]
- 活跃度：[目标]
- 转化率：[目标]
- 收入：[目标]
```

添加成功指标章节：

```markdown
## 八、成功指标

### 核心指标
- [指标 1]：[目标值]
- [指标 2]：[目标值]

### 次要指标
- [指标 3]：[目标值]
```

### 7. 更新状态

更新 frontmatter：
```yaml
stepsCompleted: ["step-01-init", "step-02-requirements", "step-03-domain-check", "step-04-domain-experts", "step-05-consolidate", "step-06-prd-draft"]
currentStep: "step-07-prd-review"
lastModified: "[当前时间]"
```

保存文档。

### 8. 确认和继续

向用户展示：

```
✅ PRD 草稿完成

已完成章节：
- 用户故事（[N] 个）
- 功能清单（[N] 个功能）
- 详细功能描述（[N] 个核心功能）
- 非功能需求
- 成功指标

下一步：PRD 审查

[A] 高级引导（详细审查）
[P] 派对模式（快速推进）
[C] 继续到步骤 07
```

等待用户输入，然后：
1. 确认 frontmatter 已更新
2. 读取 `steps/step-07-prd-review.md` 完整文件
3. 执行步骤 07
