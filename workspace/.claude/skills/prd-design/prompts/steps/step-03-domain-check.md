## 步骤目标

检查是否需要业务领域专家，并让用户选择：
1. 读取领域复杂度评估
2. 加载可用的领域专家列表
3. 根据领域过滤相关专家
4. 向用户展示选项并收集选择
5. 保存选择到 frontmatter

## 执行指令

### 1. 读取当前状态

从 PRD 文档的 frontmatter 读取：
- `domain`：业务领域
- `domainComplexity`：领域复杂度

### 2. 加载领域专家清单

使用 Read 工具读取 `domain-experts.csv` 文件。

解析 CSV 内容，提取所有专家信息：
- expert_id：专家唯一标识
- name：专家名称
- domain：适用领域
- role：角色描述
- system_path：系统路径说明
- prompt_template：提示词模板路径

### 3. 分析需求并推荐专家

**读取当前需求内容：**
- 从 PRD 文档读取"项目背景与目标"部分
- 提取关键信息：需求背景、用户痛点、目标用户、核心目标

**智能分析需求涉及的领域：**

分析需求内容，识别涉及的业务角色：
- 如果提到"买家"、"消费者"、"用户购买"、"下单" → 推荐买家域专家
- 如果提到"卖家"、"商家"、"店铺"、"商品管理" → 推荐卖家域专家
- 如果提到"服务提供者"、"服务商"、"服务能力" → 推荐服务提供者域专家
- 如果提到"平台运营"、"平台管理"、"审核"、"配置" → 推荐平台运营域专家

**生成推荐列表：**
- 推荐专家：基于需求分析识别的相关专家
- 其他专家：所有可用但未推荐的专家

### 4. 复杂度建议

基于 `domainComplexity` 给出建议：

**如果是"低"：**
```
领域复杂度：低

对于低复杂度项目，领域专家是可选的。
你可以选择跳过专家咨询，直接进入 PRD 编写。
```

**如果是"中"或"高"：**
```
领域复杂度：[中/高]

建议使用领域专家来确保：
- 识别领域特定的关注点
- 遵循行业最佳实践
- 满足合规要求
- 避免常见陷阱
```

### 5. 展示专家选项

使用 AskUserQuestion 工具向用户展示：

```
基于需求分析，推荐以下领域专家：

【推荐专家】
[1] 买家域专家
    推荐理由：需求涉及买家购买流程和用户体验
    职责：基于现有买家域功能分析需求可行性，避免功能冲突

[2] 卖家域专家
    推荐理由：需求涉及商家商品管理和店铺运营
    职责：基于现有卖家域功能分析需求可行性，避免功能冲突

【其他可用专家】
[3] 服务提供者域专家
[4] 平台运营域专家

领域专家的作用：
✓ 结合现有系统功能进行分析
✓ 识别与现有功能的冲突或重复
✓ 评估需求的技术可行性
✓ 提供符合实际业务情况的优化建议

请选择需要的领域专家（可多选，用逗号分隔，如：1,2）：

[R] 使用推荐的专家（推荐）
[A] 使用所有专家
[S] 跳过（不使用领域专家）
[数字] 手动选择（输入专家编号，如：1,3）
```

注意：
- 推荐专家列表是动态生成的，基于需求内容分析
- 推荐理由应该具体说明为什么推荐该专家
- 如果没有明确的推荐，可以展示所有专家供用户选择

### 6. 处理用户选择

**如果用户选择 [S]：**
- 更新 frontmatter：`domainExpertsUsed: []`
- 设置 `currentStep: "step-06-prd-draft"`（跳过步骤 04-05）
- 标记步骤 03 完成

**如果用户选择 [R]：**
- 使用推荐的专家列表
- 保存选择到 frontmatter

**如果用户选择 [A]：**
- 使用所有可用的专家
- 保存选择到 frontmatter

**如果用户输入数字（如"1,2"或"1,3,4"）：**
- 解析数字，获取对应的专家
- 保存选择到 frontmatter

### 7. 更新状态

更新 frontmatter：
```yaml
stepsCompleted: ["step-01-init", "step-02-requirements", "step-03-domain-check"]
currentStep: "[step-04-domain-experts 或 step-06-prd-draft]"
domainExpertsUsed: [选中的专家 expert_id 列表]
lastModified: "[当前时间]"
```

保存文档。

### 8. 确认和继续

**如果选择了专家：**
```
✅ 领域专家选择完成

已选择专家：
- 买家域专家：基于现有买家域功能分析需求
- 卖家域专家：基于现有卖家域功能分析需求

下一步：生成领域专家分析

[C] 继续到步骤 04
```

**如果跳过专家：**
```
✅ 已跳过领域专家咨询

下一步：PRD 草稿编写

[C] 继续到步骤 06
```

等待用户输入 [C]，然后：
1. 确认 frontmatter 已更新
2. 根据 `currentStep` 读取对应的步骤文件
3. 执行下一步骤
