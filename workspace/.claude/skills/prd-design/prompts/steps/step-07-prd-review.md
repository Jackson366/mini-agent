# 步骤 07：PRD 审查

## 强制执行规则（首先阅读）

### 通用规则
- 🛑 绝不在没有用户输入的情况下生成内容
- 📖 关键：在采取任何行动之前阅读完整的步骤文件
- 🔄 关键：使用 'C' 加载下一步时，确保读取整个文件
- 📋 你是引导者，不是内容生成器
- ✅ 你必须始终用中文输出

## 执行协议
- 🎯 在采取任何行动之前显示你的分析
- 💾 仅在用户选择 C（继续）时保存
- 📖 更新 frontmatter，将此步骤添加到 stepsCompleted
- 🚫 禁止在 C 被选择之前加载下一步

## 🚨 系统成功/失败指标

### ✅ 成功
- 菜单正确显示且用户输入正确处理
- 在继续之前更新 frontmatter
- 在执行之前读取整个下一步文件

### ❌ 系统失败
- 在用户未选择 'C'（继续）的情况下继续
- 未用步骤完成更新 frontmatter
- 仅读取部分步骤文件

---

## 步骤目标

全面审查 PRD 文档的质量：
1. 检查完整性
2. 检查清晰度
3. 检查一致性
4. 识别差距和歧义
5. 协作改进

## 执行指令

### 1. 读取完整 PRD

使用 Read 工具读取完整的 PRD 文档。

### 2. 完整性检查

检查以下章节是否完整：

```
完整性检查清单：

✓ 版本信息
✓ 变更日志
✓ 文档说明
✓ 项目背景与目标
  - 需求背景
  - 用户痛点
  - 目标用户
  - 核心目标
✓ 需求详情
  - 用户故事
  - 功能清单
  - 详细功能描述
  - 非功能需求
✓ 预期交付时间
✓ 成功指标
```

向用户展示检查结果：

```
完整性检查结果：

✅ 已完成：[列出完整的章节]
⚠️ 需要补充：[列出缺失或不完整的章节]
```

### 3. 清晰度检查

检查内容的清晰度：

**检查点：**
- 术语是否定义清楚？
- 功能描述是否明确？
- 业务规则是否清晰？
- 验收标准是否可验证？

向用户展示发现的问题：

```
清晰度检查发现：

⚠️ 模糊描述：
- [章节]：[问题描述]

⚠️ 未定义术语：
- [术语]：建议在"名词解释"中添加定义

建议改进：
- [具体建议]
```

### 4. 一致性检查

检查文档的一致性：

**检查点：**
- 用户故事与功能清单是否对应？
- 功能清单与详细描述是否一致？
- 成功指标与核心目标是否对齐？
- 非功能需求与业务目标是否匹配？

向用户展示发现的不一致：

```
一致性检查发现：

⚠️ 不一致：
- 用户故事 [X] 没有对应的功能实现
- 功能清单中的 [Y] 缺少详细描述

建议：
- [具体建议]
```

### 5. 差距分析

识别可能的差距：

**检查点：**
- 是否覆盖了所有用户角色？
- 是否考虑了异常场景？
- 是否有安全和隐私考虑？
- 是否有数据迁移计划（如果需要）？
- 是否有监控和运维考虑？

向用户展示差距：

```
差距分析：

⚠️ 可能的差距：
- [差距 1]：[说明]
- [差距 2]：[说明]

这些是否需要在 PRD 中补充？
```

### 6. 协作改进

与用户讨论发现的问题：

```
基于审查，我发现了以下需要改进的地方：

[列出所有问题]

你希望如何处理这些问题？

[1] 逐个讨论并修改
[2] 我自己会修改，继续下一步
[3] 这些不是问题，继续下一步
```

根据用户选择：
- 如果选择 [1]：逐个讨论并更新文档
- 如果选择 [2] 或 [3]：继续

### 7. 更新文档

如果用户选择改进，更新 PRD 文档。

### 8. 更新状态

更新 frontmatter：
```yaml
stepsCompleted: ["step-01-init", "step-02-requirements", "step-03-domain-check", "step-04-domain-experts", "step-05-consolidate", "step-06-prd-draft", "step-07-prd-review"]
currentStep: "step-08-prd-complete"
lastModified: "[当前时间]"
```

保存文档。

### 9. 确认和继续

向用户展示：

```
✅ PRD 审查完成

审查结果：
- 完整性：[评分/状态]
- 清晰度：[评分/状态]
- 一致性：[评分/状态]

已处理的问题：[N] 个

下一步：PRD 完成

[A] 高级引导（最终检查）
[P] 派对模式（快速完成）
[C] 继续到步骤 08
```

等待用户输入，然后：
1. 确认 frontmatter 已更新
2. 读取 `steps/step-08-prd-complete.md` 完整文件
3. 执行步骤 08
