# 步骤 05：整合专家分析并优化需求

## 强制执行规则（首先阅读）

### 通用规则
- 🛑 绝不在没有用户输入的情况下生成内容
- 📖 关键：在采取任何行动之前阅读完整的步骤文件
- 🔄 关键：使用 'C' 加载下一步时，确保读取整个文件
- 📋 你是引导者，不是内容生成器
- ✅ 你必须始终用中文输出

## 执行协议
- 🎯 在采取任何行动之前显示你的分析
- 💾 仅在用户选择 C（继续）时保存
- 📖 更新 frontmatter，将此步骤添加到 stepsCompleted
- 🚫 禁止在 C 被选择之前加载下一步

## 🚨 系统成功/失败指标

### ✅ 成功
- 菜单正确显示且用户输入正确处理
- 在继续之前更新 frontmatter
- 在执行之前读取整个下一步文件

### ❌ 系统失败
- 在用户未选择 'C'（继续）的情况下继续
- 未用步骤完成更新 frontmatter
- 仅读取部分步骤文件

---

## 步骤目标

与用户协作整合领域专家的分析，优化需求以符合现有系统能力：
1. 向用户展示专家分析摘要
2. 识别冲突和可行性问题
3. 协作调整需求
4. 更新 PRD 文档

## 执行指令

### 1. 读取专家分析

从 PRD 文档的 frontmatter 读取 `domainExpertOutputs`。

从文档末尾读取完整的专家分析内容。

### 2. 汇总专家分析

为每个领域专家的分析生成结构化摘要：

```
领域专家分析汇总：

【买家域专家】
✓ 可直接使用的现有功能：
  - [功能 1]
  - [功能 2]

⚠️ 发现的冲突：
  - [冲突描述]

⚠️ 需要调整的需求：
  - [需求描述]：[调整建议]

✓ 可行性评估：[高/中/低]

【卖家域专家】
✓ 可直接使用的现有功能：
  - [功能 1]

⚠️ 发现的冲突：
  - [冲突描述]

⚠️ 需要调整的需求：
  - [需求描述]：[调整建议]

✓ 可行性评估：[高/中/低]
```

### 3. 识别关键问题

汇总所有专家识别的问题：

**高优先级问题（必须解决）：**
- ⚠️ [问题 1]：[来自哪个领域] - [影响]
- ⚠️ [问题 2]：[来自哪个领域] - [影响]

**中优先级问题（建议解决）：**
- ⚠️ [问题 3]：[来自哪个领域] - [影响]

**优化建议：**
- ✓ [建议 1]：[来自哪个领域] - [预期效果]
- ✓ [建议 2]：[来自哪个领域] - [预期效果]

### 4. 协作对话

与用户讨论专家分析结果：

**问题 1：冲突处理**
"专家识别了以下与现有功能的冲突：
[列出冲突]

你希望如何处理这些冲突？
[1] 调整需求以适配现有功能
[2] 保留需求，后续评估技术方案
[3] 逐个讨论"

**问题 2：可行性问题**
"以下需求在现有系统下可行性较低：
[列出低可行性需求]

建议的调整方案：
[列出专家建议]

是否接受这些调整？"

**问题 3：功能复用**
"专家建议以下需求可以复用现有功能：
[列出可复用的功能映射]

这样可以加快实现速度，是否采纳？"

### 5. 更新 PRD 文档

基于用户的决策，更新 PRD 文档：

**5.1 添加"现有功能映射"章节**

在"五、需求详情"之前，添加新章节：

```markdown
## 4.5 现有功能映射

基于领域专家分析，以下需求可以映射到现有系统功能：

### 买家域
| 需求功能 | 现有功能 | 实现方式 | 备注 |
| -------- | -------- | -------- | ---- |
| [需求]   | [现有功能] | [复用/扩展/新增] | [说明] |

### 卖家域
| 需求功能 | 现有功能 | 实现方式 | 备注 |
| -------- | -------- | -------- | ---- |
| [需求]   | [现有功能] | [复用/扩展/新增] | [说明] |
```

**5.2 添加"冲突与风险"章节**

```markdown
## 4.6 冲突与风险

### 已识别的冲突
- ⚠️ [冲突描述]：[解决方案]

### 技术风险
- ⚠️ [风险描述]：[缓解措施]

### 可行性说明
- [领域]：[可行性评估] - [说明]
```

**5.3 更新需求描述**

根据专家建议，调整需求描述，使其更符合现有系统能力。
