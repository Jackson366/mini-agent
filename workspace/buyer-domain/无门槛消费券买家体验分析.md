## 一、买家场景分析

### 1.1 核心用户画像

| 用户类型 | 特征 | 动机 | 痛点 |
|---|---|---|---|
| **价格敏感型** | 对优惠高度敏感，会主动寻找优惠信息 | 追求性价比，希望降低决策成本 | 优惠券规则复杂、隐藏门槛多 |
| **新用户** | 首次使用平台，对平台信任度较低 | 希望低风险尝试，减少试错成本 | 不确定产品质量，怕踩坑 |
| **沉睡用户** | 曾有购买但长期未活跃 | 需要强刺激重新激活 | 缺乏购买动机 |
| **高频用户** | 经常购买，对平台较熟悉 | 把优惠券看作额外福利 | 优惠力度不够大 |

### 1.2 典型使用场景

#### 场景A：新人首次领券下单
```
用户路径：
注册成功 → 首页弹窗"新人礼包" → 领取5元无门槛券 → 浏览商品 →
加入购物车 → 结算页自动应用券 → 支付34元（原价39元）→ 收货
```

**关键体验点**：
- 券到账需要强反馈（弹窗/动画/音效）
- 首次使用时需要明确的引导："已自动为您抵扣5元"
- 支付页要突出显示"节省金额"，强化获得感

#### 场景B：老用户主动领券
```
用户路径：
打开APP → 首页Banner"限时领券" → 点击进入领券中心 →
浏览可用券 → 点击领取 → 返回继续购物 → 下单时手动选择券
```

**关键体验点**：
- 领券中心需要清晰的筛选："即将过期"/"最大面额"/"新券"
- 领取后给予即时反馈："领取成功！已到账"
- 支持按"省钱最多"智能排序

#### 场景C：券即将过期提醒
```
用户路径：
收到push"您有1张券即将过期" → 点击查看 → 看到"5元券今晚到期" →
快速浏览商品 → 找到合适商品 → 紧急下单使用
```

**关键体验点**：
- 提醒要制造紧迫感但不焦虑
- 推荐可用商品降低决策成本
- 提供倒计时增强紧迫感

## 三、转化分析

### 3.1 漏斗节点与流失原因

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              无门槛券转化漏斗                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  触达用户(100%)                                                               │
│      ↓ 券面额不够吸引 / 不信任平台 / 暂无购买需求                              │
│  点击领券(60%)                                                                │
│      ↓ 领取流程复杂 / 需要登录 / 已达领取上限                                  │
│  领取成功(55%)                                                                │
│      ↓ 找不到可用商品 / 券规则不清晰 / 忘记使用                               │
│  浏览商品(30%)                                                                │
│      ↓ 商品价格超出预期 / 不想买了 / 找不到使用入口                            │
│  加入购物车(20%)                                                              │
│      ↓ 运费太高 / 发现更便宜的替代品 / 券无法使用                              │
│  进入结算(15%)                                                                │
│      ↓ 支付环节卡顿 / 改变主意 / 发现券已过期                                  │
│  完成支付(12%)                                                                │
│                                                                              │
│  总转化率：12%                                                                │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 3.2 关键流失点与优化策略

#### 流失点1：点击领券 → 领取成功（流失5%）
**主要原因**：
- 需要登录才能领取，嫌麻烦直接放弃
- 提示"已领取过"，但实际想再领一张
- 系统卡顿，领券按钮无响应

**优化建议**：
```
✅ 快速登录：支持微信一键登录，减少填写表单
✅ 清晰提示：即使已领过，也展示"您已领过，快去使用吧"并引导到券列表
✅ 防重复领取：显示"每个用户限领1张"，但支持每日签到领新券
✅ 性能优化：领券接口响应时间 < 500ms
```

#### 流失点2：领取成功 → 浏览商品（流失25%）
**主要原因**：
- 领券后不知道去哪里用
- 找不到合适的商品
- 忘记自己有券

**优化建议**：
```
✅ 领券后引导：Toast提示"去使用"，点击跳转到商品列表
✅ 智能推荐：根据券的可用范围，推荐匹配商品
✅ 持续提醒：领券后1天未使用，push提醒"您有1张券待使用"
✅ 券列表置顶：在"我的"页显眼位置展示"我的券"
```

#### 流失点3：浏览商品 → 加入购物车（流失10%）
**主要原因**：
- 商品价格还是偏高（券抵扣后仍超预期）
- 找不到"加入购物车"按钮
- 看了商品详情后不想买了

**优化建议**：
```
✅ 价格展示：商品列表直接显示"券后价"，降低计算成本
✅ 购买引导：商品详情页突出"立减5元"标签
✅ 紧迫感营造：显示"仅剩X张"，刺激立即购买
```

#### 流失点4：加入购物车 → 进入结算（流失5%）
**主要原因**：
- 购物车未显示券的使用状态
- 不知道如何使用券
- 运费超出预期

**优化建议**：
```
✅ 购物车提示：底部显示"已用券省5元，去结算"
✅ 明确引导：结算按钮旁显示"使用优惠券抵扣"
✅ 运费透明：提前显示运费，避免结算时流失
```

### 3.3 影响转化的关键指标

| 指标 | 计算公式 | 目标值 | 监控频率 |
|---|---|---|---|
| **券领取率** | 领取成功数 / 触达用户数 | > 50% | 实时 |
| **券使用率** | 使用券下单数 / 领取成功数 | > 20% | 每日 |
| **领券后下单率** | 下单数 / 领取成功数 | > 15% | 每日 |
| **券贡献GMV** | 用券订单GMV / 总GMV | > 5% | 每周 |
| **平均用券金额** | 用券订单平均金额 | 监控趋势 | 每周 |

## 四、异常场景处理方案

### 4.1 退款场景（用户最关心的）

#### 场景1：全额退款（未发货）
```
用户情况：用5元券购买39元商品，实际支付34元，申请全额退款
```

**推荐方案**：
```
✅ 退款金额 = 实付金额 34元
✅ 券的状态 = 退回到用户账户，恢复原有效期
✅ 提示文案："退款成功，5元券已返还，有效期至X月X日"
```

**关键逻辑**：
- 券返还后，有效期从原到期时间计算（不延长）
- 如果券已过期，则不返还，提示"券已过期，无法返还"
- 退款成功后立即到账

#### 场景2：全额退款（已发货）
```
用户情况：用5元券购买39元商品，已收到货，申请全额退款
```

**推荐方案**：
```
✅ 退款金额 = 实付金额 34元
✅ 券的状态 = 不返还，标记为"已使用"
✅ 提示文案："退款成功，优惠券已使用无法返还"
```

**理由**：已享受优惠并体验商品，券不予返还是合理规则

#### 场景3：部分退款（订单包含多商品）
```
用户情况：订单包含2件商品（A商品20元、B商品19元，合计39元），
         使用5元券后实付34元，现在只退A商品
```

**推荐方案**：
```
✅ 退款金额 = A商品分摊金额 - 券分摊抵扣
   计算：A商品占比20/39=51.3%，券分摊5×51.3%=2.57元
        退款 = 20 - 2.57 = 17.43元
✅ 券的状态 = 不返还，标记为"已使用"
✅ 提示文案："退款成功，退款金额¥17.43已到账"
```

**备选方案**（更优体验）：
```
✅ 退款金额 = A商品原价 20元
✅ 券的状态 = 退回1张5元券，但有效期不变
✅ 提示文案："退款成功，5元券已返还"
```

**推荐选择**：备选方案体验更好，用户理解成本低

#### 场景4：券即将过期时退款
```
用户情况：使用券下单后，券还有2小时过期，申请退款
```

**推荐方案**：
```
✅ 退款金额 = 实付金额
✅ 券的状态 = 退回券，但不延长有效期
✅ 提示文案："退款成功，券已返还，有效期至X日X点（即将过期）"
✅ 额外提醒：如果券有效期 < 24小时，push提醒"您的券即将过期"
```

**风险提示**：用户可能通过"反复下单退款"来延长券有效期，需要风控策略

### 4.2 券过期处理

#### 场景1：正常过期
```
用户情况：券到期未使用
```

**处理方案**：
```
✅ 自动失效：到期时系统自动将券状态改为"已过期"
✅ 列表展示：在"我的券"页显示"已过期"标签，保留7天后隐藏
✅ 提前提醒：过期前1天、1小时各提醒一次
```

#### 场景2：订单中券过期
```
用户情况：用户已选择券但未支付，券过期了
```

**处理方案**：
```
✅ 实时校验：支付时再次检查券是否有效
✅ 友好提示："您的优惠券已过期，请重新选择"
✅ 推荐替代：如果有其他可用券，自动推荐新券
```

#### 场景3：过期券补领
```
用户情况：券过期了，用户想要再领一张
```

**处理方案**：
```
✅ 限制条件：如果活动仍在进行，允许再次领取
✅ 特殊规则：新人券过期后不补领（仅限新用户）
✅ 提示文案："该券已过期，您可领取其他优惠券"
```

### 4.3 并发与竞态场景

#### 场景1：多设备同时下单
```
用户情况：用户在手机APP和电脑网页同时用同一张券下单
```

**处理方案**：
```
✅ 乐观锁：用券时加锁，防止重复使用
✅ 友好提示："该优惠券已在另一笔订单中使用"
✅ 自动切换：如果有其他券，自动替换
```

#### 场景2：优惠券库存耗尽
```
用户情况：用户点击领券，但券已被抢完
```

**处理方案**：
```
✅ 实时反馈："来晚了，优惠券已被抢完"
✅ 引导关注："关注我们，第一时间获取最新优惠"
✅ 推荐替代："您可以领取其他优惠券"
```

## 五、过期与提醒策略

### 5.1 提醒渠道与时机

| 提醒时机 | 渠道 | 提醒内容 | 频率限制 |
|---|---|---|---|
| **领取成功（即时）** | Toast | "领取成功！5元券已到账" | 每次领取 |
| **领取后24小时未用** | Push | "您有1张5元券待使用，快去挑选心仪商品吧" | 1次/券 |
| **过期前1天** | Push + 站内信 | "您的5元券将于明天过期，请尽快使用" | 1次/券 |
| **过期前1小时** | Push | "紧急提醒：您的5元券还有1小时过期" | 1次/券 |
| **已过期** | 站内信 | "您的券已过期，关注活动获取新券" | 1次/券 |

### 5.2 提醒文案设计

**Push通知策略**：
```
❌ 不推荐：您有优惠券即将过期
✅ 推荐：5元券今晚到期！这些商品都能用（附商品链接）
✅ 推荐：最后1小时！您的5元券即将过期，点击使用

关键要素：
- 明确面额（5元）
- 制造紧迫感（今晚到期/最后1小时）
- 提供行动入口（点击使用）
```

**站内信策略**：
```
┌─────────────────────────────────────────┐
│ 📢 优惠券过期提醒                        │
│ ─────────────────────────────────────── │
│ 您的「5元无门槛券」将于明日过期          │
│                                         │
│ 券面额：5元                             │
│ 有效期至：2026-03-04 23:59:59           │
│                                         │
│ [立即使用] [查看可用商品]                │
└─────────────────────────────────────────┘
```

### 5.3 过期券的展示策略

**保留时长建议**：
- 过期后保留 **7天**，然后自动隐藏
- 保留期间置灰显示，标注"已过期"
- 提供"查看历史券"入口，可查看所有过期券

**展示位置优先级**：
```
1. 可用券（默认展示）
2. 即将过期券（置顶，高亮）
3. 已过期券（折叠，需点击展开）
```

### 5.4 补领与挽留策略

**场景**：用户券过期后，想要再次领取

**策略**：
```
✅ 限时补领：过期后3天内，可再次领取1张（降低用户挫败感）
✅ 签到领券：连续签到7天，送1张无门槛券（培养习惯）
✅ 生日特权：生日当月可领取专属券
✅ 沉睡唤醒：30天未下单，发放1张召回券
```

## 六、用户心智分析

### 6.1 无门槛券培养的用户习惯

#### 正向习惯
```
✅ 降低决策门槛：小金额商品也能用券，增加购买频次
✅ 即时满足感：无需凑单，立刻享受优惠
✅ 信任建立：平台直接补贴，用户感知"平台在让利"
```

#### 负向习惯（需警惕）
```
⚠️ "等券"心理：用户可能养成"不发券就不下单"的习惯
⚠️ 价格锚定：用户会将券后价作为"正常价格"，券停后不适应
⚠️ 券依赖：过度依赖券，对正价商品失去购买意愿
```

### 6.2 避免用户"等待发券"的策略

#### 策略1：券的常态化发放
```
✅ 每日签到：每天都能领小额券（1-2元）
✅ 任务奖励：完成指定任务（如分享商品）获得券
✅ 会员特权：会员每月固定发放券
```
**效果**：用户知道"随时有券可领"，降低等待心理

#### 策略2：券的差异化设计
```
✅ 无门槛券：小额（5元），高频发放
✅ 满减券：大额（满100减20），低频发放
✅ 会员券：专属大额券，培养会员身份认同
```
**效果**：用户不会期待"天天有大额券"

#### 策略3：价值传递
```
✅ 强调商品品质：引导用户关注商品本身，而非价格
✅ 会员权益对比：显示会员专享价，突出长期价值
✅ 内容营销：通过种草内容建立品牌信任
```

### 6.3 与其他券型的配合

#### 券型组合策略

│                           券型组合矩阵                                 │

│                                                                      │

- 明确告知用户是否允许叠加
- 系统自动计算最优方案

## 八、给 Main 的交付建议

### 8.1 核心结论

1. **退款场景是用户最关心的**，建议采用"未发货返还券，已发货不返还"的简单规则
2. **过期提醒直接影响券使用率**，必须提前1天+1小时双提醒
3. **自动最优推荐能显著降低决策成本**，推荐作为默认行为
4. **券后价展示能有效刺激转化**，建议在商品列表就展示

### 8.2 需要业务方确认的问题

| 问题 | 选项 | 影响 |
|---|---|---|
| **退款券是否返还**？ | A. 未发货返还，已发货不返还<br>B. 任何情况都不返还 | 影响用户体验和成本 |
| **券是否允许叠加使用**？ | A. 允许与满减券叠加<br>B. 不可叠加 | 影响订单计算复杂度 |
| **过期券保留多久**？ | A. 保留7天<br>B. 永久保留 | 影响列表性能和用户习惯 |
| **过期前多久提醒**？ | A. 提前1天<br>B. 提前3天<br>C. 提前1天+1小时 | 影响券使用率和打扰度 |

### 8.3 与其他域专家的协同点

#### 与 platform-operations 协同
```
- 券的投放节奏：建议"每日签到"常态化发放，避免用户等券心理
- 成本控制：建议设置"每用户限领1张"，防止被薅羊毛
- 效果评估：建议监控"券使用率"和"领券后下单率"
```

#### 与 seller-domain 协同
```
- 成本分摊：建议平台全额补贴，降低商家参与门槛
- 商家自主权：建议允许商家选择"是否参与无门槛券活动"
- 券使用范围：建议先支持"全场通用"，后期支持"指定商家"
```

#### 与 ui-expert / ux-expert 协同
```
- 券的视觉设计：建议使用红色/橙色等暖色调，强化促销感
- 领券动画：建议设计"金币飞入账户"的动画，增强获得感
- 券列表交互：建议使用标签页分类（可用/即将过期/已过期）
```

### 8.4 潜在风险预警

| 风险 | 描述 | 缓解措施 |
|---|---|---|
| **用户薅羊毛** | 批量注册账号领券 | 设备指纹+IP限制+实名认证 |
| **券被黄牛倒卖** | 账号+券一起倒卖 | 券不可转赠+实名绑定 |
| **成本失控** | 券使用率过高，超预算 | 设置总发行量+实时监控 |
| **用户等券心理** | 只在发券时下单 | 常态化发券+价值传递 |
| **退款漏洞** | 反复下单退款延长券有效期 | 未发货才返还券+风控规则 |

### 8.5 建议的MVP范围

**第一期（MVP）**：
- ✅ 用户领券（主动领取）
- ✅ 券的使用（自动最优推荐）
- ✅ 订单金额扣减
- ✅ 券状态管理（可用/已用/过期）
- ✅ 基础过期提醒（提前1天）
- ✅ 简单退款返还（未发货返还，已发货不返还）
- ✅ 我的券列表

**第二期（体验优化）**：
- 券的分类展示（可用/即将过期/已过期）
- 多券选择交互
- 券后价展示
- 智能推荐商品
- 券使用记录

**第三期（高级功能）**：
- 券分享裂变
- 券转赠功能
- 券积分兑换
- 券自动发放（触发器）
